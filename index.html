Improving Page Load Times Optimize UX and Maximize Profits

M odern technological advances have given our generation the capacity to get things done immediately. With this expectation of instant gratification, users are generally more impatient, and aren't inclined to wait around while their websites load. In fact, Amazon found that a delay of as little as 100ms could lead to a 1% decrease in sales. Other surveys also show that slow website load time was the main factor in shoppers forgoing a purchase.

Discoverability is another factor to consider. Google now takes into account page load speed when generating rank search results, making faster website more discoverable. It is also imparative to consider the importance of page load times to enhance user experience. By implementing the appropriate strategies as outlined below, we can increase the speed at which pages load, improve user experience, and potentially boost revenue and discoverability for businesses. Let's get started!

1. Optimize Images

As attractive as they may be, the fact is that images are heavy. On average, images account for roughly 65% of a websiteâ€™s byte size, and can be an obstacle for delivering your content smoothly. To make sure they are not slowing down your users, be conscious of their sizes. Some ideas:

  • Pick the right format (JPG, PNG, GIF etc.) for your image
  • Eliminate unnecessary background images, and replace them with CSS effects
  • Serve up scaled images that match the user's browser size

2. Place JS Files at the Bottom of the Page

If your Javascript code is in a separate file, place the script for the file at the bottom of the page, before the close of the body tag. This is a quick and simple solution that allows the page content to load before your Javascript does. This means that users will not have to wait for the Javascript to load before getting to the actual content of the page.

3. Minify Your Code

Another way to decrease page load times is to optimize your HTML, CSS, and JS files. This can be done by minifying, in which white space, comments, new lines, and indentations are removed. There are many tools out there which can help you minify your files. Simply paste the website's URL, or input the code directly. Here are some resources:

  • Minifier
  • Closure Compiler

4. Make Use of Caching

Caching refers to the storage of static files, such as images and objects, in the user's hard drive. This allows pages to load faster, as the files do not need to be retireved each time it is requested. There are plugins that can be installed to enable caching (if you're using WordPress).

  • W3 Total Cache
  • WP Super Cache
  • WP Rocket

5. Use External Scripts Sparingly

External scripts send HTTP requests every time a page is loaded, slowing down a website's load time. Eliminating the following scripts may help to deliver your content more smoothly.

  • Bootstrap
  • External web fonts
  • Icon packs like Font Awesome
  • Third party commenting systems

6. Check Your Speed

Finally, check how fast your website loads! Resources like Google's PageSpeed Tools can give you a rating of your page's load time, as well as tips on how to enhance your website's overall performance. Just paste the URL to your website, and see how your score improves as you make changes.

Lastly, it's important to note that not everyone has access to high speed internet. Many users around the world are still on a 3G network or worse. By making efforts to decrease page load times and enhance website performance, we can create pages that are easily accessible to all!
